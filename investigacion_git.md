# Investigación Git Básico

¿Qué es el "staging area" o "índice" en Git y para qué sirve? 



Es una sona de git donde se preparan  los cambios que iran a tu proximo commit 



&nbsp;



Que es un working directory, staging area y repository 



Working directory:es el espacio en tu dispositivo donde ese encuentran los archivos de el proyecto en el que se trabaja actualmente 



Que es string area: es un area intermedia donde donde puedes seleccionar los cambios espesificos del working directory  que quieres mantener para la proccima confirmacion  



&nbsp;



Que es un repocitorty : es el espacio dentro del git que guarda el historial completo en foma de confirmaciones  



&nbsp;



2\) que es un git status: git satatus permite un analisis de lo que pasa actualmente con el working directory 



El usarlo con frecuencia permite mantenerse al tanto de todo lo que pasa y tenter un control claro de lo que esta pasando 



&nbsp;



3\) . Diferencia entre git fetch y git pull 



Git fetch. descarga los cambios del repositorio a tu repositorio local pero no los integra a la rama de trabajo actual 



&nbsp;



Git pull. Hace el trabajo de git fetch y luego los fuciona automaticamente en tu rama de trabajo 



&nbsp;



4\) ¿Qué es un "merge conflict" y cómo se resuelve? 



Ocurre cuando dos ramas de desarrollador an modificado la linea en el mismo archvo a sido modificada en una y alterada en otra y git no sabe cual es la vercion correcta
